资源库

模块加载时使用的构建资源需要在资源库中注册，每个模块都拥有自己的资源库

属性 类型 描述
style Object CSS 文件资源
script Object javascript 文件资源
source Object 模版文件资源
data Object 数据源数据资源

一个完整的资源库例子

export default { // 模块的资源库（对内／对外） resources: { // js 资源 script: { main: "main.js" }, // 模版资源 source: { index: "index.html", footer: "frameworks::footer" }, // css 资源 style: { common: "frameworks::common", main: "frameworks::docs", style: "css/main.css" }, // 数据资源 data: { list: "./list.json", // 远程数据地址，并且配置参数，参数从模块参数配置中解析 user: "http://api.ioing.com/getUser|@param(id:id)" } }, config: { ...config } }

资源仓库中如果对应的资源类型不存在，该类型则可缺省

清晰的资源管控

资源库的目的是提高代码的可读性，其能够被可知该模块的所有引用资源，另外该模块资源库中的资源也能够共享给其他模块。资源库能够有效避免开发者在代码逻辑中引用资源文件，从而构建一个简单明了的模块模型

URI 引用规则

资源引用的 URL 默认相对路径为模块根目录

符号 例 描述
./ 或 空 ./test.json 或 test.json 当前模块根目录
/ /moduleA/a.css modules 根目录
:: moduleA::name 指向模块 A 下的同类型资源

下面是一个基本示例

export default { resources: { style: { common: "frameworks::common", // 指向主模块[frameworks]的样式资源[common] main: "frameworks::docs", // 指向主模块[frameworks]的样式资源[docs] style: "css/main.css" // 当前模块目录css文件夹下main.css } }, config: { ...config } }

如果在 #配置资源 中配置引用了一个模版资源，而该资源在该模块的资源库中并未进行配置，此时会尝试匹配主模块frameworks的资源库

export default { resources: { style: {}, // 空空如也 source: { 'index' // 只有一个模版 } }, config: { ... style: ['common'], // 尝试到主模块的仓库中查找样式表资源 'common' source: ['index', 'header', 'footer'] // 尝试到主模块的仓库中查找模版资源 'header' 和 'footer' data: ['setting'] // 尝试到主模块的仓库中查找数据资源 'setting' } }

根据该应用特性可省略对模块frameworks的指向引用

注意多层级依赖

如果使用::的形式引用其他模块资源时，如果被引用模块尚未被加载，则会自动分析依赖关系加载被依赖的模块配置。此方法是为了引用复用率高的资源，但应注意避免设置多层级依赖资源

引用类型

支持的引用类型有三种：String 类型、Object 类型、Function 类型

String 类型

表示一个 URL 地址 或 内部指令（即上一张卡片中所讲到的引用规则）

Object 类型

直接返回一个 Object 对象

export default { resources: { data : { // 数据源 book 为一个直接返回的 Object 对象 book : { name : "道德经", title : "道可道，非常道。名可名，非常名。" } } }, config: { ...config } }
注意

直接返回 Object 对象时需要注意的是该对象被修改时会影响之后所有该模块的副本，如果你不想将这个改变传递给后续模块副本可以是使用匿名函数将对象抛出

export default { resources: { data : { // 数据源 book 为一个直接返回的 Object 对象 book : function () { return { name : "道德经", title : "道可道，非常道。名可名，非常名。" } } } }, config: { ...config } }

技巧：多个模块引用同一个直接返回的 Object 资源时能够利用改变传递特性进行模块间数据通信。比如登录状态，如果多模块通过引用登录状态对象来渲染模版，当这个状态在任何模块中被更新时都能及时响应到所有引用模块

Function 类型

通过函数返回以上两种类型，返回后的类型同样符合对应应用类型的解析规则，比如该函数执行后返回一个字符串，那么该字符串会被识别为一个新的 URL 路径解析即重新执行 String 类型逻辑，如果该函数返回的是一个 Object 类型数据则直接返回获取数据，如果该函数返回为undefinde时该函数则被认为是一个异步返回值函数，会等待函数 callback 结果

export default { resources: { data : { // 数据源 book 为一个直接返回的 Object 的函数 book : function () { return { name : "道德经", title : "道可道，非常道。名可名，非常名。" } }, // 数据源 book1 返回了一个字符串，这将被视为一个新的解释规则 book1 : function (param, callback) { return "http://api.ioing.com/getBook/" + param.id } // 数据源 book2 执行了一个 Ajax 异步函数，通过 callback 返回值 book2 : function (param, callback) { fetch("http://api.ioing.com/getBook/" + param.id).then(function (err, data) { callback(data) }) } } }, config: { ...config } }

函数类型默认会接收两个参数[param, callback]，第一个参数为当前模块的参数对象，第二个参数为 callback 函数，callback 接收一个参数即返回值

URL 解析规则

URL 类型除了具备上述卡片中 String 类型的基本引用规则外还具备一些深入的规则指令，指令通过关键符号@来声明，每个特性间通过|来进行分隔，主要的特性包括：参数规则

@method 请求类型

指令 值 描述
@method() GET | POST | JSONP 设定请求类型为 GET 或 POST 方式
export default { resources: { data : { // 此数据源以 POST 方式进行请求 book : "http://api.ioing.com/getBook|@method(POST)" } }, config: { ...config } }
注意

缺省此设置项时默认以 GET 方式请求数据

@param 请求参数

指令 值 描述
@param() key:value 设定参数的名称和值
export default { resources: { data : { // 获取 id 为100的数据列表，翻页为第一页，当前页数量为20 book : "http://api.ioing.com/getBook|@param(id:id,page:page,limit:20)" } }, config: { ...config // 模块默认参数 param : { id : 100, page : 0 } } }

上例数据源[book]的请求 URL 实际解释为：http://api.ioing.com/getBook?id=100&page=0,limit=20

有的时候服务端有一些奇怪的传参需求，比如参数为类似 JSON 的格式

export default { resources: { data : { // 获取 id 为100的数据列表，翻页为第一页，当前页数量为20 book : "http://api.ioing.com/getBook|@param(request:{id:id,page:page,limit:20})" // 这更像是一个Json的参数，但这种写法与上面缺省外面花括号的写法实际结果相同 book1 : "http://api.ioing.com/getBook|@param({request:{id:id,page:page,limit:20}})" } }, config: { ...config // 模块默认参数 param : { id : 100, page : 0 } } }

上例数据源[book]的请求 URL 实际解释为：http://api.ioing.com/getBook?request={'{'}"id":100,"page":0,"limit"=20}

注意

@param() 里的参数和 JSON 类似（可以缺省外面的花括号），key 值是字符串不被解析，value 值为对象可被解析，其可以解析与模块参数同名的变量

@cache 数据源的缓存时间

指令 值 描述
@cache() timeout 秒 设定缓冲数据源超时时间
export default { resources: { data : { // 设定该数据源缓存超时时间为60妙 book : "http://api.ioing.com/getBook|@param(id:id,page:page,limit:20)|cache(60)" } }, config: { ...config // 模块默认参数 param : { id : 100, page : 0 } } }

上例中如果该数据源在被请求后的60秒内进行第二次请求时该数据源不会真的发起请求而是直接抛出上一次缓存数据

@storeage 数据本地常驻时间

指令 值 描述
@storeage() timeout 秒 设定本地存储数据源超时时间
export default { resources: { data : { // 设定该数据在本地存储有效期为一个月 book : "http://api.ioing.com/getBook|@param(id:id,page:page,limit:20)|storeage(2592000)" } }, config: { ...config // 模块默认参数 param : { id : 100, page : 0 } } }

对于有些更新不敏感的数据（比如城市数据）可以使用 @storeage 指令将其常驻于浏览器本地存储

@origin 取消 "withCredentials" 认证

指令 值 描述
@origin() true 设定该 url 为不需要带 'withCredentials' 认证的请求

默认情况下如果被请求的 URL 与当前域存在跨域，引擎将自动为请求带上 'withCredentials' 认证，以保证跨域请求，但如果是 CDN 上的静态资源，则服务器一般将允许源设置为*以匹配所有网站，如果我们此时设置认证则会报错，可通过使用 @origin(true) 指令设定忽略域，以取消跨域认证

知识点补充

跨域请求静态资源时 CDN/服务器的 Response Headers 一般会设置 Access-Control-Allow-Origin:*，此时如果为跨域请求自动带上 'withCredentials' 认证则会被拒绝，如果服务器返回的设定不是 '*' 而是你当前请求域名则可以通过，但出于安全考虑一般都会使用 '*' 指定，此时请求不会传送 cookie

指令 值 描述
@storeage() timeout 秒 设定本地存储数据源超时时间

@headers 设定请求headers

指令 值 描述
@headers() key:value 设定 headers 的名称和值
export default { resources: { data : { // 设置请求链接的 headers book : "http://api.ioing.com/getBook|@headers(timeout:'3600s',id:id,page:page)" } }, config: { ...config // 模块默认参数 param : { id : 100, page : 0 } } }

和 @param 的参数规则一样，@headers 也能解析参数中的同名变量

@settings 发起请求的设定

指令 值 描述
@settings() key:value 设定 settings 的名称和值
export default { resources: { data : { // 设置请求链接的 settings book : "http://api.ioing.com/getBook|@settings(contentType:'application/x-www-form-urlencoded')" } }, config: { ...config // 模块默认参数 param : { id : 100, page : 0 } } }

和 @param 的参数规则一样，@settings 也能解析参数中的同名变量

@caller JsonP 跨域请求

设定 @caller 跨域参数时需要先设定 @method(JSONP) 才能有效

指令 值 描述
@caller() keyName | 空 请求时传递随机函数名参数的约定 key 值
export default { resources: { data : { // JsonP 的实际请求链接为："http://api.ioing.com/getBook?callback=__call__[随机数]" book : "http://api.ioing.com/getBook|@method(JSONP)|@caller(callback)" } }, config: { ...config } }

跨域请求数据还有一个简便的写法："http://api.ioing.com/getBook?callback=~"，其中"=~"会被识别为跨域请求标志，符号"~"将被替换为随机函数名

export default { resources: { data : { // 此链接也会被识别为 JsonP 请求，其实际请求链接为："http://api.ioing.com/getBook?callback=__call__[随机数]" book : "http://api.ioing.com/getBook?callback=~" } }, config: { ...config } }

有时候你的服务器可能并不支持 JsonP 服务，比如一些 CDN 服务器，此时为了支持静态资源的跨域引擎提供了另一套体系，首先你需要不设置 @caller 的任何参数，就像这样： @caller(), 其次你需要改写你的资源文件

以 main.css 为例看一下改写前文件

body{ text-align:center; overflow:hidden; overflow-y:hidden } ...

改写后的文件

// main.css 远程静态跨域文件 style("main", "body{text-align:center;overflow:hidden;overflow-y:hidden}...")

style 是当前文件的类型，我们可以理解为它是通过 JsonP 加载回来执行的函数，该函数第一个参数则是该函数资源库中的 key 名称， 而第二个参数是该文件的文本内容。我们称这种类型的文件为：静态化 JsonP 格式文件

如果你需要支持其他类型文件的静态化 JsonP 格式，则需要使用对应的函数名来包裹

style、source、data 分别可以定义资源库中的三大类资源文件，如果你需要更多的文件类型方法可使用 define 进行封装

包裹方法 参数 示例
style() key,context style("main", "...css")
source() key,context source("index", "...html")
data() key,context data("list", "...json")
define() type,key,context define("xml", "list", "...context")
