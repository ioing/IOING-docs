模块类型

模块是构成应用的主要元素，也是应用内容展示的载体，模块能够在应用中独立运行且可以在应用中自由装卸

模块分为 #主模块 和 #子模块，主模块一般承载了应用程序的公共部分，如视图中的 header 和 footer 部分以及其它公共组件部分，子模块一般承载了具体的功能场景，如“消息界面”和“通讯录界面”等

子模块又分为复合型模块（也称嵌合型模块）和独立型模块（即全屏模块），复合型模块是被嵌入在主模块视图中显示的，独立性模块是全屏显示的

应用模块的意义

传统单页面中我们所有的使用场景都被堆放在同一个 DOM Tree 下，于是造成了高耦合的问题，比如模块之间相互影响以及 CSS 样式和 Javascript 变量的重名覆盖，而沙盒化能够将模块页面限制在沙盒（Sandbox）中，使其与主 DOM Tree 脱离
