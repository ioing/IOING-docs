STEP 1：了解项目文件

克隆 IOING 项目到本地，通过事例进行学习

克隆整个文档示例

git clone https://github.com/ioing/IOING

或纯净开发版

git clone https://github.com/ioing/IOING-DEV
项目结构

默认情况下模块的开发在modules目录中进行，该目录中必须包含一个主视图模块frameworks。在模块目录中创建一个新文件夹并以模块名命名，该文件夹即为这个新模块的开发目录

克隆完成后，打开项目目录，并安装依赖包

cd [项目目录] sudo npm install

使用自动创建模块来替代 STEP 2-3 步骤，使用该命令能快速创建一个示例模块

gulp new --module [模块名]

同样也可以快速创建一个组件示例

gulp new --components [组件名]

STEP 2：配置你的模块

模块目录中必须包含一个配置文件config.js

配置文件用以描述模块的基本特性和资源库，比如模块的运行模式、转场动画、生命周期等#了解更多模块配置

为模块配置模版、数据、样式#了解更多资源配置

export default { // 模块的资源库（对内／对外） resources: { // 模版资源 source: { index: "index.html" }, // 模块样式 style : { main : "main.css" }, // 模块数据 data: { lang : ["你好！", "Hello!", "こんにちは!", "안녕 하세요!", "bonjour!", "Hallo!", "Ciao!", "Hola!", "Здравствуйте!"] } }, // 关于模块特性的配置 config : { leve : 1, absolute : true, background : "#fff", // 需要加载的模版 source: ["index"], // 需要加载的样式 style : ["main"], // 引用数据源 data: ["lang"] } }

每个模块都是由配置中的 HTML, CSS, Js 资源所构成的一个独立 WEB 页面，和一般页面不同的是通过配置组装的模块页面能够互不干扰的共存

为什么通过配置引用资源？

通过配置引用能够使依赖资源和业务逻辑分离。对于数据型资源能够进行自动配参获取，从而避免在业务逻辑中使用大量 Ajax/fetch 等来取得数据#了解资源规则

什么是主模版？

主模版就是资源配置 source 数组中序列为 0 的模版，该模版将作为模块入口被载入，其它模版则作为引用资源模版#了解资源配置
